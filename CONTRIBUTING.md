# Contributing Guidelines

## Contract-First Rule

Prima di scrivere codice:

1. Verificare i file in /contracts
2. Non modificare v1
3. Se necessario creare /contracts/v2

## Versioning

- v1 = stable
- v2+ = breaking changes

## Security

Non committare:
- Chiavi private
- File license.json runtime
- Variabili .env

## AI Usage

Qualsiasi AI utilizzata per sviluppo deve:
- Ricevere i contratti ufficiali
- Non alterare gli endpoint definiti
- Non modificare naming convenzioni
